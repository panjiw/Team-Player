Changes made to the SRS:

  • Cleaned up formatting for the Product Description and Process Description document in the SRS. Aspects included fixing bullet points and inconsistent line breaks. Additionally, fixed typos and poor wording problems.

  • Updated the Product Description to describe repeating tasks, which were not outlined before hand. Additionally, updated product descriptions to better describe finalized details on specific aspects of tasks and bills- I.E. adding details on cycling bills.

  • Added Asana as an additional competitor, mainly because our rescoping incorporated group task management.

  • Updated the software toolset. Because we’re using Ruby on Rails, we updated SQLite3 as an internal database that we won’t be writing queries for. Additionally, we added Bootstrap, Jenkins, and Heroku as toolsets for our product.  

  • Updated UI Diagrams based off of paper prototype assignment
  • Refined views to make it easier to transition
  • Now able to view bill summary before individual bills
  • Decided to hide filter to allow more calendar space.

Users can only filter by group; they cannot further filter tasks and bills. The ToDo List will assist them in viewing their own events.

  • Decided to added a help page accessible from every page, in addition to help popups for new users.

  • Updated open source calendar. Now using Angular object calendar instead of FullCalendar (jQuery plugin)

  • Switched Keith and Fung’s primary roles. Keith has experience with databases and will focus on back end development. Switch Fung to frontend.

  • New project timeline and plans. Based off of the last two weeks, we learned how much time certain parts of the project may take. Getting started took a while, but we were able to still finished the login before the zero feature release, as planned. Having a better understanding of Ruby on Rails allows us to speed up creating models and controllers, but learning Angular JS will be a bit more difficult. We also realized the importance of front and back end integration, so planned certains to ensure this is discussed and works.
