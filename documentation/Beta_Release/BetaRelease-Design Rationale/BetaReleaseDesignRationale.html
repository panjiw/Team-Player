SRS Changes:

  • Updated UI Diagrams based off of paper prototype assignment
  • Refined views to make it easier to transition
  • Now able to view bill summary before individual bills because seeing the total first is more convenient to user
  • Decided to hide filter to allow more calendar space.
  • Users can only filter by group; they cannot further filter tasks and bills. The ToDo List will assist them in viewing their own events.
  • Updated the software toolset. Because we’re using Ruby on Rails, we updated SQLite3 as an internal database that we won’t be writing queries for. Additionally, we added Bootstrap, Jenkins, and Heroku as toolsets for our product.  
  • Added Asana as an additional competitor, mainly because our rescoping incorporated group task management.
  • Decided to added a help accessible from every page, in addition to help popups for new users.
  • Changed the word “teams” to “groups”
  • We had thought that since our app was called TeamPlayer, we could have a bunch of teams that contained players. However, it sounded too much like a sports team to our customers, and felt that it was more natural to call groups of people “groups” instead.
  • Tasks can have a due date or no due date
  • Sometimes, task just need to be done, and don’t necessarily have a particular deadline. These will be called “todo’s”.
  • External documentation or help guide is currently a question and answer list. We will implement an intro step by step tutorial when time allows. However, when users first sign in, they will be able to directed to immediate add a new group.

SDS Changes:

  • Included more design changes such as: self groups, bill un-cycling, overdue task constraints, new task generation and un-inheritance of tasks and bills
  • See SDS for tradeoffs
  • Updated productivity calendar to make sure we create a user tutorial, as well as make sure we meet with customers and TA to test and go over each release.
