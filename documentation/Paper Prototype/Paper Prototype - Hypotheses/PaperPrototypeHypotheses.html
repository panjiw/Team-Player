1. “Dateless” tasks: We need feedback on whether “tasks” without dates make sense. They will be displayed in a separate bar on the top left, but we need to see whether it is clear what they are if they don’t appear on the calendar
  2. Task Creation Window: Our task/event creation screen includes: name, description, date/time or no date/time, selecting which group and members within the group, making it a cyclic event (cycles through selected group members), and/or making it a repeating event (like every Monday, etc.). This UI has a lot of components, and we need to see if this is 1. not too many pieces, and 2. clear what each piece means. This is especially true of the “cyclic” events.
  3. Filters: We currently have a sidebar that filters what group’s information you are viewing on the calendar. However, for simplicity, we decided there would be only one bar--so if you only selected one of three groups to view in the calendar, then switched to the bills view, you still would only see that one group’s bills. We’re curious both to see whether it makes sense to keep filters the same between the views, and whether people will actually use the filters (i.e. are they even that important).
